# Contributing to Crypto AI Advisor

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Focus on the code, not the person
- Help others learn and grow
- Report concerns to maintainers

## How to Contribute

### 1. Report Bugs

Create an [Issue](https://github.com/yourusername/crypto-ai-advisor/issues) with:
- Clear title
- Detailed description
- Steps to reproduce
- Expected vs actual behavior
- Environment info (Node version, OS)

### 2. Suggest Features

Create an [Issue](https://github.com/yourusername/crypto-ai-advisor/issues) with:
- Clear title
- Use case and motivation
- Proposed solution
- Why it's useful

### 3. Submit Code

1. **Fork** the repository
2. **Create branch**: `git checkout -b feature/your-feature`
3. **Make changes** following code style (see DEVELOPMENT.md)
4. **Write tests** for new features
5. **Update docs** if needed
6. **Commit**: `git commit -m "feat: your feature"`
7. **Push**: `git push origin feature/your-feature`
8. **Open PR** with:
   - Clear title and description
   - Links to related issues
   - Screenshots if UI changes
   - Testing instructions

## Pull Request Checklist

- [ ] Code follows style guide
- [ ] Tests pass: `npm run test`
- [ ] Linting passes: `npm run lint`
- [ ] Type checking passes: `npm run type-check`
- [ ] Build succeeds: `npm run build`
- [ ] Documentation updated
- [ ] No breaking changes (or documented)
